③ src/train.py PyTorch
Training script · sklearn breast_cancer dataset
Trains a binary classifier from scratch each run, logs every hyperparameter and per-epoch metric to MLflow, registers the model, and saves the training distribution as the drift reference baseline.

② FASTAPI :8000 API Gateway

④ src/monitor.py Evidently
Monitoring script · invoked via POST /run-monitoring
Hourly job that fetches the 500 most recent predictions from PostgreSQL and runs Evidently's KS-test-based drift detection against the training distribution, saving an HTML report browsable via FastAPI.
Load current predictions (SQLAlchemy) rows = load_recent_predictions(limit=500) if len(rows) < 30: write monitoring_skipped.txt; return current_df = pd.DataFrame([row["features"] for row in rows]) current_df["prediction_probability"] = [row["probability"] for row in rows] current_df = current_df[reference_df.columns] # align column order Reference snapshot reference_df = pd.read_parquet(REFERENCE_FILE) Evidently drift report report = Report([DataSummaryPreset(), DataDriftPreset()]) evaluation = report.run(current_data=current_df, reference_data=reference_df) evaluation.save_html(REPORTS_DIR / "data_drift_latest.html") evaluation.save_json(REPORTS_DIR / "data_drift_latest.json") evaluation.save_html(REPORTS_DIR / f"data_drift_{timestamp}.html")
⑤ MLflow :5001 Tracking+Registry
Experiment tracking · model versioning
Persistent store for every training run's parameters and metrics, plus a versioned model registry so the API always promotes and loads the highest version number automatically.
Server startup (shell) mlflow server \ --backend-store-uri postgresql+psycopg://postgres@postgres:5432/mlops \ --default-artifact-root file:///app/mlruns \ --host 0.0.0.0 --port 5001 FastAPI loads model via MLflow client client = MlflowClient(tracking_uri=MLFLOW_TRACKING_URI) versions = client.search_model_versions(f"name = '{MODEL_NAME}'") latest = max(versions, key=lambda mv: int(mv.version)) model = mlflow.pytorch.load_model(f"runs:/{latest.run_id}/model") model.eval()
⑥ PostgreSQL :5432 mlops db
Shared DB · two consumers · two schemas
Shared relational store with two distinct roles: MLflow's metadata backend (auto-managed) and the app's prediction log table (written by FastAPI on every /predict call, read by monitor.py hourly).
prediction_logs table (src/db.py) Table("prediction_logs", metadata, Column("id", Integer, primary_key=True, autoincrement=True), Column("timestamp", DateTime, server_default="CURRENT_TIMESTAMP"), Column("features", JSON, nullable=False), Column("prediction", Integer, nullable=False), Column("probability", Float, nullable=False), ) metadata.create_all(engine) # called via init_db() on API startup conn.execute(prediction_logs.insert().values(...)) # write path SELECT … FROM prediction_logs ORDER BY timestamp DESC LIMIT 500 # read
Evidently AI Drift Reports
Data Drift + Data Quality analysis
Runs the KS test on each of the 30 features plus the prediction probability column to detect whether the live request distribution has statistically drifted from the training baseline.
Evidently report engine from evidently import Report from evidently.presets import DataDriftPreset, DataSummaryPreset # DataSummaryPreset → per-feature stats (mean, std, missing, dist.) # DataDriftPreset → KS test per feature; drift_detected: True/False report.run( current_data=current_df, # 500 rows from prediction_logs reference_data=reference_df # X_train snapshot )

⑧ ref.parquet File
./data/reference/
Snapshot of the training input distribution written once per training run; serves as the statistical baseline that Evidently compares every hourly batch of live predictions against.
Columns: X_train (30 features) + prediction_probability (float) + prediction (int) Written: reference_df.to_parquet(REFERENCE_FILE) Read: pd.read_parquet(REFERENCE_FILE) Also: mlflow.log_artifact(REFERENCE_FILE, "monitoring_reference")
⑨ ./reports/ Output
Monitoring output · filesystem
Filesystem folder where Evidently saves HTML drift reports that are immediately served by FastAPI as a static site, making drift history browsable without a separate dashboard.
data_drift_latest.html ← always overwritten data_drift_latest.json ← machine-readable version data_drift_<timestamp>.html ← archive per run app.mount("/reports", StaticFiles(directory=REPORTS_DIR))
